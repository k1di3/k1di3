What is the window object

In the browser, all global JavaScript lives inside the window object, is like the main container of everything the page can do.

Examples

For example:

alert(1)

In reality it is:

window.alert(1)

For instance if as an attacker we gain access to the window object we could execute arbitrary JS. This is awesome if we are in a sandboxed enviroment.

Aliases for window

There are several aliases for the window object:

  • self
  • top
  • parent
  • frames
  • globalThis
  • document.defaultView

Notes about these aliases:

  • self refers to the current context and is usually the same as window.
  • parent is the parent iframe, if we are not in an iframe then parent === window.
  • top refers to the highest-level window, inside nested iframes, top points to the main page window.
  • document.defaultView stores a reference to the current window object and works only when you have a document object.

Example:

document.defaultView === window // true

So we could call alert by doing this :

document.defaultView.alert(1)

This is perfect if the code blocks stuff like this : 

window.eval
window.alert

There is a trick to obtain the document object from a DOM node to then access the defaultView property . We can use a property called ownerDocument to get the document object used by the DOM node.

let node = document.createElement('div');
node.ownerDocument.defaultView.alert(1)

Using an event object

Another way to obtain the window object is by an event object.

When something happens in the browser (like a click or an error), the browser creates an event object.

In Chrome, some events include:

event.path

It is an array of elements that the event passed through.

[event.target, ..., document, window]

The last element is usually the window object, so if we use .path with .pop() the last element of the array will be returned:

<image href=1 onerror=event.path.pop().alert(1)>

Chrome has since removed the path property on the event object. The good thing is we can use the standardised composedPath() method below that will do the same shit. 

<image href=1 onerror=event.composedPath().pop().alert(1)>

Usually, each handler adds an argument with the event object except for SVG elements, this ones will add evt , so we could use this payload:

<svg><image href=1 onerror=evt.composedPath().pop().alert(1)>

Stack trace trick

In chrome, if you use the Error.prepareStackTrace callback on an Error object, you can customize stack traces, we could use this to get access to the window object. The idea here is you provide a function that has two arguments one for the error message and the other for an array of CallSite objects . A CallSite object consists of properties and methods related to the stack trace. And how we could achive window object ? Well, Chrome provide us with CallSite.getThis() , a method for CallSite objects that will return a window object if there's no `this` defined.

Error.prepareStackTrace = function(error, callSites) {
  callSites.shift().getThis().alert(1);
};
